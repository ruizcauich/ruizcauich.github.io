Cómo aprender un lenguaje de programación.

Ruiz Cauich Augusto Neftalí July 26, 2018

Este, se podría decir, que es el primer post que hago para aportar un poco de mí en temas de programación y tecnología. Me parece que es un buen tema para iniciar.

  • Primer paso: elegir el lenguaje.
  • Segundo paso: estructura básica y extensión.
  • Tercer paso: Ejecutar.
  • Cuarto paso: Tipos, variables/constantes, operadores, entrada y salida.
  • Quinto paso: Decisiones, repetición.
  • Sexto paso: Estructuras de datos y funciones.
  • Séptimo paso: Orientación a objetos, manejo de errores/excepciones.
  • Octavo paso: Otras características.

Existe en toda la comunidad las interrogantes sobre qué lenguaje es mejor, cual es que se debe escoger para aprender a programar pero ese no es el objeto de este post. Y aún así tengo que decir la búsqueda por la respuesta a esas no tiene sentido.

Sin duda lo más importante para programar es saber analizar un contexto (cual quiera que sea) y los aspectos que intervienen en él (cuales quiera que sean) y con ello aplicar un algún proceso para dar una solución (de software).

Como sea, este articulo se titula “cómo aprender un lenguaje de programación”, y por ello supondré que quien se ha llegado aquí ya sabe de programación: es un estudiante, practicante, autodidacta, etc; Que quiere aprender un lenguaje distinto al que le han enseñado o ha aprendido.

Tal vez has tratado con JAVA, C/C++, Pythons, y la lista puede ir aumentando. Sin importar tu nivel de conocimientos, espero que tengas las habilidades para resolver algún problema. Sin más, empecemos y antes, aclaro que todo esto según mi experiencia.

Primer paso: elegir el lenguaje.

Sí. Lenguajes de programación existen muchos, el consejo más fiel que puedo ofrecer es que tenemos que enfocarnos en aquel/aquellos que se usen en el área de desarrollo que nos interese: web frontend, web backend, móviles, escritorio,ciencia de datos, etc. O bien, escoger alguno que te llame la atención, si de curiosidad se trata, hay muchas alternativas.

Pero considero que lo que sigue es una buena guía para aprender cualquiera que sea.

Segundo paso: estructura básica y extensión.

Todo lenguaje de programación tiene una estructura básica para correr el programa más simple de la historia: “hola mundo!”. Bueno, podrías pensar que es tonto, que eso es solo en la universidad, colegio, o como sea. Pero es simple y quiero ser más práctico, entonces procederé ejemplificando con C, Java y Python.

Como decía líneas arriba, todo lenguaje tiene un mínimo de líneas para ejecutar un programa básico, y también, cada uno reconoce una exención para su código fuente:

C/C++: HolaMundo.c 

#include <stdio.h>

int main(){
    printf("Hola mundo!");
    return 0;
}
JAVA: HolaMundo.java 

public class HolaMundo{
    public static void main(String[] args){
        System.out.println("Hola mundo!");
    }
}
Python: HolaMundo.py 

print(“Hola Mundo!”)

Bueno, ¿y qué sigue?. Entender por supuesto que significa cada línea de código (no te estreses por ello en este post). Si vamos con C/C++ (comparten la estructura básica) podemos ser breves y definir dos secciones donde la primera es en la que incluimos librerías que se usarán en el programa, luego las funciones. C/C++ son lenguajes compilados, el compilador de estos busca la función main para determinar el punto de inicio de ejecución del programa. Y dentro las instrucciones necesarias.

Java es muy similar, la diferencia y/o característica es que todo debe estar dentro de una clase (si no sabes que es una clase, es un concepto relacionado con la orientación a objetos, de nuevo, no te estreses). Dentro de la clase, una función que al igual que en C/C++ es aquella que se busca para ejecutar.

Python, muy sencillo, ¿no es así?.

Puedo resumir aun más: en general la estructura de estos es primero importar las librerías a usar, y luego, escribir el código necesario para dar entrada a la ejecución.

A demás, quiero incluir en este paso los comentarios, importantes para documentar el código. Debemos buscar cómo escribir un comentario en nuestro lenguaje.

Tercer paso: Ejecutar.

Claro, ¿cómo vas a usar un programa si no lo ejecutas?. En este punto es importante que sepas que existen lenguajes compilados y lenguajes interpretados y por lo tanto para ejecutar un programa se debe seguir un proceso distinto.

Para los lenguajes de ejemplo es necesario abrir la consola de línea de comandos de nuestro sistema y situarnos en el directorio donde el archivo está. Y lo que sigue, te lo dejo.

Cuarto paso: Tipos, variables/constantes, operadores, entrada y salida.

Los datos son importantes en la programación, estos son de un tipo de dato u otro: enteros, flotantes, cadenas de caracteres, números complejos, etc. Entonces, en este paso debes aprender a definir una variable de algún tipo, y esto lo pongo de la mano con obtener datos del usuario, realizar alguna operación y presentar los resultados.

Seré breve en esta ejemplificación, pues ya sabes realmente cómo va la estructura de tu lenguaje.

Java: 

import java.util.Scanner; // En la primera línea, antes de public class…
// Dentro de main, obtiene un entero, lo multiplica e imprime el resultado
Scanner entrada = new Scanner(System.in);
int numero = entrada.nextInt();
int numeroPorDos = numero*2;
System.out.println( numeroPorDos );
Python: 

numero = int(input(“numero: ”))
numero2 = numero*2
print(numero)

Quinto paso: Decisiones, repetición.

Decidir hacia donde se debe dirigir el flujo de ejecución del programa es el siguiente paso. Saber como se hace un if o switch, ciclos como for, while y do-while. NO todos los lenguajes usan esos tres tipos de ciclo, como python, que tampoco tiene una instrucción switch. Sin embargo, eso no le resta poder a este lenguaje.

Sexto paso: Estructuras de datos y funciones.

Hasta este punto los conocimientos básicos del lenguaje como la sintaxis del mismo ya han sido practicada. Aun así apenas sabes crear variables de tipos numéricos, booleanos, cadenas; lo que sigue es saber agrupar valores: arreglos, litas, diccionarios/mapas hash, estructuras, conjuntos, tupas, etc.

A demás, consideró que en este momento es importante empezar a entender cómo el lenguaje maneja la memoria y almacena la información. Esto es importante pues de esa forma entenderemos mejor lo que ahora sigue.

Las funciones y cómo compartir datos entre ellas. Definir la función y los parámetros que necesita para que haga su labor y precisamente hablando de los parámetros es donde se involucra el manejo de la memoria y las variables, si pasamos por alto esto podríamos modificar información en el programa sin saberlo entonces, el resultado del programa no será en esperado.

Séptimo paso: Orientación a objetos, manejo de errores/excepciones.

La orientación a objetos es un tema amplio. Saber como definir una clase, un objeto, sus métodos, cómo se define la herencia, si además de clases existe otro concepto que maneja el lenguaje como java y las interfaces.

Posterior a ello llega el manejo de errores, que si bien lo he situado en esta sección, es un tema que puede aprenderse desde el cuarto paso.

Octavo paso: Otras características.

Hay mucho por aprender sobre un lenguaje, uno es ,por ejemplo, el manejo de archivos que se puede aprender desde el paso cuatro, pues también es un método de entrada y salida. Sin embargo esta aquí pues bien puede ser fácil implementar o todo lo contrario,puede ser complejo. Todo depende del lenguaje de programación.

Otro aspecto es la conexión con bases de datos, y la manipulación de sus registros: obtención, inserción, eliminado, modificado, manejo de metadatos, etc.

Y por último, el aspecto: desarrollo de interfaces visuales, eventos.

En termiteros generales, estos pasos te ayudaran en el aprendizaje de un nuevo lenguaje y es aplicable al menos para todos aquellos lenguajes comúnmente conocidos. Pero no te quedes solamente con esto, busca alguna otra opinión que de sobra hay y ten en cuenta que podrías encontrarte alguna tecnología dependiente de otra o que se usan en conjunto como JavaSript y HTML.
